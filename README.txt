В базе данных заранее загружены два теста для демонстрации работы программы. 
При создании, редактировании и прохождении тестов программа понимает как указание одного правильного ответа, так и множественный выбор. В случае, если у вопроса есть несколько правильных ответов, он зачтется отвеченным правильно только если пользователь выбрал все правильные ответы. При необходимости можно продолжить работу над программой и добавить проверку, предполагающую засчитывание балла, если пользователем указан хотя бы один из правильных ответов. 
При создании теста допускается произвольное количество вариантов ответа от 2х до 4х. Первые два ответа обязательны для заполнения, остальные два - по желанию. Они не отключаются, чтобы иметь возможность добавить на их место другой вариант при редактировании теста. 
Также при создании теста пользователю недоступны поля ввода вопроса и ответов, пока не будет указано название нового теста. Так же реализована и вкладка редактирования теста. Там эти поля будут неактивны, пока не будет выбран тест из списка существующих.
При создании добавлены проверки на пустой вопрос, пустые первые два варианта ответов, отсутствие казанного правильного ответа и указание правильным пустого ответа. 
При удалении теста из базы данных удаляется как информация о тесте, так и его вопросы. 

В программе реализован паттерн MVVM. Некоторые модели остались пустыми, поскольку им было достаточно базовых моделей, общих для нескольких моделей представления, такие модели собраны в папке Models/Core. 
Из всех представлений соответствующая модель представления отсутствует только у QuestionCard (тип элемента UserControl), поскольку это не самостоятельный элемент, а дочерний для модели представления CompleteTestVM, который вызывается только ею и делит данные с ней. 

Было использовано несколько собственных стилей для оформления текста, вкладок, кнопок и иконок, для удобства все они собраны в папке Styles.

Для читаемости кода все методы, обращающиеся к базе данных, в других методах указаны с DBAccess.<Название_метода>, соответствующее пространство имен не подключалось (мне это кажется более удобным, но если это не сочетается с стилем кода компании - также можно исправить).

Многое можно было бы улучшить, в ограниченных временных рамках сделала всё, что смогла. Буду благодарна любой обратной связи по этому заданию.
